# Homepage Experience — v3.4 Audit and Migration Notes

## Audit findings

The previous homepage had a strong hero but then moved directly into a
three-project gallery, a resource preview, and social profiles. This created
three problems: it repeated project discovery, did not make the learning
direction visible, and did not create a clear path from evidence to
collaboration. Writing and AI Lab content were absent from the narrative even
though the platform already supports both concepts.

## Narrative sequence

1. Hero: identity, direction, and first action.
2. Current Focus: current work as evidence, not metrics.
3. Featured Project: one complete reason to inspect a case study.
4. Learning Journey: completed, current, and next topics.
5. Latest Writing: an honest Medium destination while published article data is
   not yet available.
6. Resources: concise, useful references.
7. AI Lab: a labelled preview of planned work.
8. Collaboration: an explicit contact and GitHub close.

## Content integrity

No reading time, article title, project result, or AI capability has been
invented. The writing panel states that technical writing is in progress; a
future content integration may replace it with a verified Medium article.

## Migration notes

`PortfolioHub` remains the homepage composition boundary, but its former
project-grid, resources, and profile sections have been replaced with a
purposeful sequence. Existing project, resource, and AI Lab data are reused.
No API, routing, admin, localization provider, or SEO code is changed.
